    计算机组成原理 Chapter 5：存储层次、Cache 与虚拟内存

    这一章的主线是 memory hierarchy：CPU 很快，主存和磁盘相对慢得多，所以计算机系统需要用分层结构把“快但贵的小容量存储”和“慢但便宜的大容量存储”组合起来。

    可以把本章分成三层理解：

    • 存储器层次：为什么要用 SRAM、DRAM、Flash、磁盘分层。
    • Cache：怎样利用局部性让处理器多数时候访问更快的存储。
    • 虚拟内存：怎样让主存像磁盘的 Cache，并为程序提供独立的地址空间。

    存储器的基本类型

    随着处理器速度快速提升，内存访问速度没有以同样速度增长，CPU 经常需要等待数据。存储系统的设计目标就是在成本可接受的前提下，尽量降低平均访问时间。

    常见存储器可以按速度、价格和容量大致分层：

    类型 是否易失 典型用途 特点
    SRAM 易失 CPU 内部 Cache 很快、价格高、容量小
    DRAM 易失 主存 RAM 容量较大、需要刷新、速度低于 SRAM
    Flash 非易失 SSD、U 盘 断电保存、读写速度介于 DRAM 和磁盘之间
    Magnetic Disk 非易失 机械硬盘 容量大、价格低、机械访问延迟高

    DRAM

    DRAM 是动态随机存取存储器。它用电容中的电荷表示数据，但电荷会逐渐泄漏，所以必须周期性刷新。刷新会带来额外开销，因此 DRAM 比 SRAM 慢。

    DRAM 的读取通常是破坏性的，读出后需要把数据重新写回。一般说的“内存条”主要就是 DRAM。现代 DRAM 引入时钟信号和流水化组织，因此也常见 SDRAM 这类名称。

    SRAM

    SRAM 是静态随机存取存储器。它用多个晶体管构成锁存结构，只要持续供电，数据就能保持，不需要像 DRAM 那样刷新。

    SRAM 的速度快，但面积和成本高，所以通常集成在 CPU 内部作为 L1、L2、L3 Cache。

    Flash 与磁盘

    Flash 是非易失性存储，断电后数据仍然保留，常用于 SSD 和 U 盘。

    机械磁盘由盘片、磁道、扇区和磁头组成：

    • track：磁道，盘片表面的同心圆。
    • sector：扇区，磁道上的一段，是磁盘常见的最小物理读写单元。
    • cylinder：柱面，多张盘片上半径相同的磁道集合。

    磁盘访问时间通常由以下部分组成：

    \[ Disk\ Access\ Time = Seek\ Time + Rotational\ Latency + Transfer\ Time + Controller\ Overhead \]

    例如：512B sector、15000rpm、平均寻道 4ms、传输率 100MB/s、控制器开销 0.2ms。

    平均旋转延迟约为半圈：

    \[ \frac{0.5}{15000 / 60} = 0.002s = 2ms \]

    传输 512B 的时间非常小，约为：

    \[ \frac{512B}{100MB/s} \approx 0.005ms \]

    所以总时间大约为：

    \[ 4ms + 2ms + 0.2ms + 0.005ms \approx 6.2ms \]

    这说明机械磁盘的主要开销来自寻道和旋转延迟，而不是实际传输那几百字节的时间。

    存储器层次结构

    越靠近 CPU 的存储器速度越快、容量越小、价格越高；越远离 CPU 的存储器容量越大、价格越低、速度越慢。

    典型层次如下：

    1. Registers
    2. L1 Cache
    3. L2/L3 Cache
    4. Main Memory
    5. SSD / Disk

    存储层次结构依赖 局部性原理：

    • 时间局部性：刚被访问的数据，短时间内很可能再次被访问。
    • 空间局部性：某个地址附近的数据，很可能很快也会被访问。

    在任意相邻两层之间，数据以固定大小的单位复制。这个单位通常叫 block 或 line。一个 block 往往包含多个 word，所以找到 block 后，还需要用 offset 指出具体访问 block 内的哪一个字节或字。

    Hit、Miss 与平均访问时间

    如果处理器需要的数据已经在高层存储中，称为 hit。如果高层没有找到，需要去低层取回，称为 miss。

    常见概念：

    • hit time：命中时访问高层存储所需的时间，也包括判断是否命中的时间。
    • miss penalty：缺失后，从低层取回 block、填入高层并把数据交给处理器的时间。
    • miss rate：访问中发生 miss 的比例。

    平均访问时间可以写成：

    \[ AMAT = Hit\ Time + Miss\ Rate \times Miss\ Penalty \]

    这也是 Cache 设计中非常重要的公式。即使命中时间很小，如果缺失率或缺失代价很大，整体性能仍然会很差。

    Cache 的作用

    Cache 位于 CPU 和主存之间，用较小但更快的 SRAM 保存近期可能会被访问的数据。它的目标不是保存所有数据，而是利用局部性让“大多数访问都命中”。

    引入 Cache 后，核心问题变成：

    1. 一个内存块应该放在 Cache 的哪里？
    2. CPU 访问某个地址时，怎样判断它是否在 Cache 中？
    3. 如果 Cache 满了，应该替换哪个块？
    4. 写操作应该什么时候同步到低层存储？

    直接映射 Cache

    Direct Mapped Cache 是最简单的一种组织方式：每个内存块只能放到 Cache 中唯一的一行。

    映射关系为：

    \[ Index = Block\ Address \bmod Number\ of\ Cache\ Blocks \]

    如果 Cache 行数是 2 的幂，取模就等价于取地址中的若干低位作为 index。

    一个地址通常被拆成三部分：

    字段 作用
    Tag 区分映射到同一行的不同内存块
    Index 指出访问 Cache 的哪一行
    Offset 指出在 block 内访问哪一个字节或字

    Cache 行中还需要保存 valid bit：

    • valid = 0：这一行还没有有效数据。
    • valid = 1：这一行保存着可用数据，需要继续比较 tag。

    直接映射 Cache 的访问流程可以写成：

    1. 用 index 找到唯一的 Cache 行。
    2. 检查 valid bit 是否为 1。
    3. 比较保存的 tag 和地址中的 tag。
    4. 如果 valid 为 1 且 tag 匹配，则 hit。
    5. 否则 miss，需要从低层存储取回对应 block。

    直接映射的优点是硬件简单、访问快；缺点是冲突 miss 明显。两个不同内存块如果 index 相同，就会反复互相替换。

    Block Offset 与字节寻址

    如果一个 block 中包含多个 word，那么 offset 需要指出 block 内部具体位置。

    以 64 位系统为例，如果一个 word 是 4B，那么从 word offset 转成 byte offset 还需要 2 位，因为：

    \[ 4B = 2^2B \]

    如果 block 中有 \(2^m\) 个 word，那么选择 word 需要 \(m\) 位；再选择 word 内的字节需要 2 位，所以 byte offset 共有：

    \[ m + 2 \]

    这也是很多 Cache 地址拆分题容易出错的地方：题目到底是在说 word offset，还是 byte offset。

    虚拟内存

    继续沿着层次结构思考：既然 Cache 可以把主存当作低层存储，那么主存也可以作为磁盘的 Cache。

    这就是虚拟内存的基本思想：

    Main memory can act as a cache for secondary storage.

    程序看到的是 Virtual Address，也就是虚拟地址。虚拟地址不是数据在真实内存条上的位置。操作系统和硬件会把虚拟地址转换成 Physical Address，再访问主存。

    虚拟内存带来的好处包括：

    • 让每个程序看起来拥有连续、独立的地址空间。
    • 保护不同进程的内存互不干扰。
    • 允许程序使用比物理内存更大的地址空间。
    • 让不常用的页暂时放在磁盘中。

    Page 与 Page Fault

    在虚拟内存中，block 通常称为 page。虚拟地址被拆成两部分：

    • Virtual Page Number，虚拟页号。
    • Page Offset，页内偏移。

    页内偏移的位数决定页大小。例如页内偏移是 12 位，则页大小为：

    \[ 2^{12}B = 4096B = 4KiB \]

    如果访问的虚拟页不在物理内存中，就会发生 page fault。Page fault 类似 Cache miss，但代价大得多，因为可能需要访问磁盘。

    一次 Page Fault 的典型处理过程：

    1. CPU 发现页表项无效，触发异常。
    2. 操作系统接管控制权。
    3. 操作系统在磁盘或交换区中找到需要的页。
    4. 如果物理内存已满，选择一个页换出。
    5. 如果被换出的页是 dirty 的，需要写回磁盘。
    6. 把目标页读入物理内存。
    7. 更新页表和相关状态位。
    8. 重新执行刚才失败的指令。

    因为一次缺页可能花费数百万个时钟周期，所以虚拟内存设计通常采用：

    • 较大的 page，常见为 4KiB 或更大。
    • 更灵活的放置策略，通常接近全相联。
    • 软件处理缺页，因为磁盘开销远大于软件处理开销。
    • 写回策略，用 dirty bit 避免每次写都同步磁盘。

    页表

    页表是虚拟页号到物理页号的映射表。每个进程通常有自己的页表，CPU 中会有页表寄存器指向当前进程页表的起始位置。

    地址翻译流程如下：

    1. CPU 产生虚拟地址。
    2. 用虚拟页号 VPN 查询页表。
    3. 检查页表项中的 valid bit。
    4. 如果 valid 为 1，取出物理页号 PPN。
    5. 将 PPN 与原来的 page offset 拼接成物理地址。
    6. 如果 valid 为 0，触发 page fault。

    例如，48 位虚拟地址、4KiB 页大小时：

    • Page offset 为 12 位。
    • VPN 为 36 位。

    如果物理地址是 40 位，则：

    • Page offset 仍为 12 位。
    • PPN 为 28 位。

    翻译时 offset 不变，只替换页号部分。

    为什么页表会很大

    如果虚拟地址是 48 位，页大小是 4KiB，则 VPN 有 36 位，也就是说理论上页表可能有：

    \[ 2^{36} \]

    个条目。

    如果每个页表项占 8B，仅一张线性页表就可能需要：

    \[ 2^{36} \times 8B = 512GiB \]

    这显然不可接受。因此实际系统会使用多级页表，只为真正使用到的虚拟地址范围分配页表页。

    Page Replacement 与 Dirty Bit

    当发生 page fault 而物理内存没有空闲页时，操作系统必须选择一个页换出。理想目标是换出未来最久不会被访问的页，但未来无法准确知道，所以常用过去行为进行近似。

    LRU，也就是最近最少使用策略，基于一个假设：很久没被访问的页，短期内再次访问的概率较低。

    写回策略中，每个页通常有 dirty bit：

    • dirty = 0：页没有被修改，换出时可以直接丢弃。
    • dirty = 1：页被修改过，换出前必须写回磁盘。

    这样可以减少不必要的磁盘写入。

    TLB

    TLB 全称是 Translation Lookaside Buffer，可以理解为页表查询的 Cache，用来加速虚拟地址到物理地址的转换。

    如果每次访存都要先访问内存中的页表，再访问真正的数据，那么一次内存访问会变成至少两次内存访问。TLB 保存最近使用过的 VPN 到 PPN 的映射，可以显著降低地址翻译开销。

    TLB 访问流程：

    VA = VPN + offset

查 TLB
  Hit:
    PPN + offset -> PA
  Miss:
    查页表
      页有效:
        更新 TLB
        PPN + offset -> PA
      页无效:
        Page Fault
        从 Disk 调页
        更新页表和 TLB
        重新访问

    小结

    Chapter 5 可以抓住这几个关键词：

    • 存储器层次结构用小而快的上层存储弥补大而慢的下层存储。
    • Cache 依赖时间局部性和空间局部性。
    • 直接映射 Cache 用 index 决定唯一位置，用 tag 判断是否匹配。
    • 虚拟内存让主存成为磁盘的 Cache，同时提供隔离和保护。
    • Page fault 代价很高，所以页较大、替换策略更复杂、常由软件处理。
    • TLB 是页表查询的 Cache，用来加速地址翻译。
